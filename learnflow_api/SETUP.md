## 🏠 1. ตั้งค่า Local Development

### Step 1: Clone & Install Dependencies
```bash
cd learnflow_api
pip install -r requirements.txt
```

### Step 2: Configure Environment
```bash
# Copy .env.example เป็น .env
cp .env.example .env

# แก้ไข .env ด้วยค่าของคุณ
```

**ค่าที่ต้องแก:**
```
DB_HOST=localhost          # MySQL server
DB_USER=root              # MySQL user
DB_PASSWORD=...           # MySQL password
FIREBASE_CREDENTIALS=config/serviceAccountKey.json  # Firebase key file
FLASK_DEBUG=true          # Enable debug mode
```

### Step 3: Setup MySQL Database
```bash
# 1. Create database
mysql -u root -p
> CREATE DATABASE learnflow CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
> exit;

# 2. Import schema
mysql -u root -p learnflow < database/schema/main_schema.sql
mysql -u root -p learnflow < database/seeds/sample_data.sql
```

### Step 4: Download Firebase Credentials
1. ไปที่ Firebase Console: https://console.firebase.google.com
2. Select your project → Project Settings
3. Service Accounts → Generate New Private Key
4. Save as `config/serviceAccountKey.json`

### Step 5: Run Local Server
```bash
python -m flask --app app run --debug
```

✅ Server runs at: **http://127.0.0.1:5000**

## 🌐 2. Deploy ให้คนอื่นใช้ (Online)

### Option A: Railway.app (แนะนำ - ง่ายที่สุด)

#### Step 1: Prepare Repository
```bash
# ในโฟลเดอร์ root ของ project
git add .
git commit -m "Initial commit: LearnFlow API"
git push origin main
```

#### Step 2: Deploy on Railway
1. ไปที่ https://railway.app
2. Sign up → New Project
3. Connect GitHub repository
4. Select branch: `main`
5. Configure environment variables ใน Railway:
   ```
   MYSQLHOST=<auto-generated>
   MYSQLPORT=3306
   MYSQLUSER=<auto-generated>
   MYSQLPASSWORD=<auto-generated>
   MYSQLDATABASE=learnflow
   
   FIREBASE_CREDENTIALS_JSON={"type":"service_account",...}  # Paste full JSON
   FLASK_DEBUG=false
   CORS_ORIGINS=https://yourdomain.com,https://app.yourdomain.com
   ```

#### Step 3: Run Database Migration
Railway จะ auto-deploy หลังจาก git push

ให้รัน migration script ครั้งแรกเท่านั้น:
```bash
railway run python database/init_db.py
```

✅ Your API is live at: **https://your-app.railway.app**

---

### Option B: Heroku (วิธีเดิม)

```bash
# 1. Install Heroku CLI
# 2. Login
heroku login

# 3. Create app
heroku create learnflow-api

# 4. Set environment variables
heroku config:set DB_HOST=... DB_USER=... DB_PASSWORD=...
heroku config:set FIREBASE_CREDENTIALS_JSON='{"type":"service_account",...}'

# 5. Deploy
git push heroku main
```

---

### Option C: Self-hosted VPS (Nginx + Gunicorn)

#### Step 1: SSH to Server
```bash
ssh user@your-server-ip
```

#### Step 2: Install & Setup
```bash
git clone <your-repo-url>
cd learnflow_api
pip install -r requirements.txt
```

#### Step 3: Configure Systemd Service
```bash
# Create /etc/systemd/system/learnflow-api.service
[Unit]
Description=LearnFlow API
After=network.target

[Service]
Type=notify
User=www-data
WorkingDirectory=/var/www/learnflow_api
ExecStart=/usr/bin/python3 -m gunicorn -w 4 -b 0.0.0.0:5000 "app:create_app()"
Restart=always

[Install]
WantedBy=multi-user.target
```

#### Step 4: Start Service
```bash
sudo systemctl start learnflow-api
sudo systemctl enable learnflow-api
```

## ⚠️ 5. Common Issues & Fixes

### Issue: "ModuleNotFoundError: No module named 'firebase_admin'"
```bash
pip install -r requirements.txt
```

### Issue: "Can't connect to MySQL server"
```bash
# Check MySQL is running
mysql -u root -p -e "SELECT 1"

# Or if not running
sudo service mysql start  # Linux
brew services start mysql  # macOS
```

### Issue: "FIREBASE_CREDENTIALS file not found"
```bash
# Download from Firebase Console
# Save to: config/serviceAccountKey.json
```

### Issue: "CORS error when calling from Flutter"
```
Edit .env:
CORS_ORIGINS=http://localhost:3000,https://yourdomain.com
```
